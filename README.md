# Prueba Técnica - Product Dashboard

## Contexto
Esta aplicación fue desarrollada rápidamente para una demo interna.
Funciona correctamente pero necesita mejoras antes de pasar a producción.

## Tu tarea
Tienes **máximo 2 horas** para mejorar el código según tu criterio.
No esperamos que arregles todo - queremos ver cómo priorizas y tu approach al refactoring.

## Instalación
```bash
npm install
npm run dev
```

La app estará disponible en http://localhost:3000

## API
La aplicación usa Fake Store API (https://fakestoreapi.com) para los datos de productos.

## Qué puedes hacer
- Refactorizar código
- Mejorar la estructura
- Añadir/mejorar TypeScript
- Optimizar rendimiento
- Corregir bugs
- Mejorar la UX/UI
- Añadir tests (si lo consideras prioritario)
- Actualizar dependencias
- Usar cualquier herramienta que prefieras (IA incluida)

## Entrega
1. Haz los cambios que consideres necesarios
2. Documenta tus decisiones en un archivo IMPROVEMENTS.md
3. Envíanos el código en un ZIP con:
    - El código mejorado
    - El archivo IMPROVEMENTS.md explicando:
        - Qué problemas identificaste
        - Qué cambios hiciste y por qué
        - Qué harías con más tiempo

## Notas importantes
- Prioriza cambios con mayor impacto
- El código debe seguir funcionando

## Criterios de evaluación
- Capacidad de identificar y priorizar problemas
- Calidad del refactoring
- Claridad en la documentación de decisiones
- Pragmatismo vs over-engineering

¡Buena suerte!
